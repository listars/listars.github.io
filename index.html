Summary-Interview

前言

​ 这两周，工作上的事情开始汇报成周报，以前会把一些想法也一起总结到周总结，但是周报写这些感觉不太妥当，从这周开始换个地方输出自己的一些心路历程或者一些学习的小插曲

面试

​ 最近，主要进行技术面试比较多，有一些值得反思，吐槽 😓 和总结的地方：

反思

​ 一直以来我的面试风格都是面带微笑，不管面试者的强弱，表现好坏，我都一视同仁，保持微笑～～～这样的原因是我怕有些面试者会比较紧张，放不开。其实这一招还是有点效果的，有遇到过很多一开始不是很放得开的面试者，但慢慢的就放松下来了。

​ 但是！！！还是有部分容易得意忘形的人，其实交流不过 5 分钟我就已经不想要他了，问题基本全错，但是我还保持着微笑，而这些人觉得面试官带着笑容就是满意的意思吧，越来越觉得自己十分 ok。这就是我为何要反思的原因了，我以后还是保持随时切换无表情的做法可能更好一些。只有当面试者紧张的时候才微笑让他们放松

吐槽

  • 有时宁愿少也不想多

有时拿到一份简历，基本过一眼就觉得老样子模版，XXX 技术精通，熟练，写了一堆，XXX 项目写了一堆。但实际交流过程都是见仁见智的，既然你写了，我就默认你是会的，但是回答不上来或者达不到内心的预期我觉得就是减分，项目再多，不如一两个能体现自己能力所在的项目，概述清楚「当然，你要是真的很牛逼就当我说的都是废话啦 😝」

  • 希望能更加自信的“销售”自己

其实一个面试过程最多半小时左右，我觉得这半小时要让我单方面去挖掘一个人的优点是很难的，一问一答式的面试其实在我看来是很普通的面试，这也意味着很大概率是在面试一个普通的人。

要如何做到让自己在面试官眼里看来是特别的呢：

  • 往往面试官问的每个问题都是可延伸性的，如果你觉得还有什么是你理解的，可以继续延伸补充。⚠️ 点到为止
  • 一般我都会问面试者觉得自己有什么优势，但是我没问到的「我觉得这是个很关键的问题，因为我觉得这半小时我没法完全了解你，我希望你能自我表达自己的优势，以达到加分效果」，如果这时候回答没有，不知道或者含含糊糊的基本都是减分操作

总结

​ 面试，不要太带入个人情绪，应该更多站在公司和团队的角度去看待面试者，还要以岗位的标准去衡量，这还真是一门学问

结语

​ 今年也有个小目标，看 vue 2.x 的源码还有学习 canvas，一直以为都觉得 canvas 的碰撞检测很牛逼，以前看过一本 canvas 的书让我感觉很刺激「碎碎念：希望能有 canvas 大神过来面试」～～～其实之前有考虑过学 java，但是以目前来说，我想看源码和学 canvas 的兴趣更大点 😄

btoa/atob
  • btoa 方法可以将 ASCII 字符串和二进制数据转换成一个 base64 编码后的字符串
  • atob 方法可以将 base64 编码后的字符串进行解码

那这样是否意味着我们不可以使用 btoa/atob 对中文进行 base64 编码/解码了么？

答案当然不是，先让我们了解一下几个编码的方法再走进黑魔法的世界

escape/unescape
  • Unescape 方法会把传入的十六进制编码值返回 ASCII 字符串
  • Escape 方法会生成新的由十六进制转义序列替换的字符串

当你想需要对 URI 进行操作时，请无视这两个方法

encodeURI/decodeURI
  • 这两个方法目的是为了对 URI 进行完整的编码/解码
